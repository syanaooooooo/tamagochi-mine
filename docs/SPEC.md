## 一、项目概览

### 1.1 产品定位

一款以像素风格为主的独立虚拟宠物 Web App。用户可以养育、照顾、繁殖有独特外观的小宠物，通过日常互动建立情感连接。以手机端传播为主，桌面端体验同步迭代。

### 1.2 核心体验目标

- **陪伴感**：开着屏幕就能看到宠物一直在动，有生命感
- **观察感**：宠物状态变化肉眼可见，玩家能感知到照顾的效果
- **情感张力**：宠物会「生病」「变悲伤」，但不过度惩罚，保留合理容错
- **轻度策略**：进化路线因照顾方式不同而不同，给玩家真实的掌控感

### 1.3 设计原则

- 每日签到制为主节奏，不强迫实时在线
- 宠物睡眠期间状态消耗极慢，减少焦虑
- 死亡后可继承上一代基因，延续情感连接
- 货币系统数值精心平衡，保证挑战感和公平感

## 四、成长系统

### 4.1 成长阶段

```
🥚 蛋  ──→  🐣 幼年（固定形态）  ──→  🐦 成年（2-3种形态，由照顾方式决定）
```

- **蛋**：孵化时间约 24 小时（从创建账号/选择宠物开始计算，次日登录即可孵化）
- **幼年**：固定外观，持续约 3-5 天（实际天数）
- **成年**：进入后外观固定，持续至死亡或繁殖

### 4.2 进化路线（每只角色 3 种成年形态）

进化由幼年期结束时（第4天）根据两个隐性指标快照决定，**对玩家不可见**（不显示数值，靠直觉养成）：

| 指标 | 累计方式 | 说明 |
|------|---------|------|
| **照顾失误次数** | +1/次 | 饥饿归零、便便脏一天、健康归零未喂药超过半天 |
| **训练得分** | +1/次点击训练 | 每天上限 +5，防刷 |

**进化判定逻辑：**
```
失误 ≤ 2 且 训练 ≥ 12  →  珍稀形态
失误 ≥ 6               →  野性形态（无论训练多少）
其余                    →  普通形态
```

## 五、核心状态系统

### 5.1 三条状态值

| 状态 | 范围 | 归零后果 |
|------|------|---------|
| 🍗 **饱食度** | 0-100 | 开始消耗健康值 |
| 😊 **心情** | 0-100 | 开始缓慢消耗健康值 |
| ❤️ **健康** | 0-100 | 归零 = 宠物死亡 |

### 5.2 状态消耗速率

**清醒状态（白天）：**
- 饱食度：-5/小时
- 心情：-3/小时
- 健康：自然 -1/小时，额外损耗见下

**睡眠状态（用户设定的睡觉时段）：**
- 所有消耗速率 ÷ 5（极慢）
- 睡眠中若健康 > 50，轻微恢复 +1/小时

**级联伤害：**
- 饱食度 = 0：健康额外 -5/小时
- 心情 = 0：健康额外 -2/小时
- 生病状态（便便引起）：健康额外 -3/小时

### 5.3 典型日常周期（示例）

```
晚 10pm 宠物睡觉 → 消耗极慢
早 8am 用户起床打开 App → 饱食度约 -10（可管理）
早上喂食 → 饱食度恢复
白天上班 → 宠物白天自然消耗
傍晚回家互动 → 补充心情、清理便便
```

## 六、照顾系统

### 6.1 全部互动动作（MVP 全部实现）

| 动作 | 效果 | 备注 |
|------|------|------|
| 🍚 **喂食（正餐）** | 饱食度 +35 | 每天大约需要 2-3 次 |
| 🍬 **喂零食** | 饱食度 +5，心情 +20 | 不能代替正餐 |
| 🎮 **玩耍（进入 Mini-game）** | 心情 +20-40（随得分），赚货币 | 每日可玩多次 |
| 💊 **喂药** | 健康 +30 | 生病时使用 |
| 🚽 **清理便便** | 消除「脏」状态 | 拖到第二天会生病 |
| 🛁 **洗澡** | 健康 +10，心情 +10 | 每日一次上限 |
| 🏋️ **训练** | 心情 -10，训练得分 +1 | 影响进化路线 |
| 🌙 **设定睡眠时间** | 设定后进入低消耗模式 | 可每日调整 |
| 🤲 **抚摸** | 心情 +10 | 无冷却，随时可做 |

### 6.2 便便机制

- 宠物白天会随机产生便便（约每 4-6 小时一次）
- 同一天内不清理不会直接生病
- **当天零点结算**：若当日便便未全部清理 → 宠物进入「脏」状态
- **第二天零点结算**：若脏状态持续未处理 → 触发「生病」状态
- 生病时健康持续下降，需要喂药治疗

### 6.3 宠物情绪状态（影响 Idle 动画）

| 状态条件 | 对应 Idle 动画 |
|---------|--------------|
| 所有值 > 70 | 开心蹦跳 / 做鬼脸 |
| 饱食度 < 30 | 肚子咕噜、可怜眼 |
| 心情 < 30 | 耷拉脑袋、叹气 |
| 健康 < 30 | 晕眩星星、发抖 |
| 生病 | 体温计、冒汗 |
| 有便便未清 | 捏鼻子 / 绿烟雾效果 |
| 睡觉 | 闭眼、ZZZ 气泡 |
| 被抚摸中 | 眯眼 + 爱心 |

## 九、货币经济系统

### 9.1 货币来源

| 来源 | 数量 | 频率 |
|------|------|------|
| 每日签到 | +20 💰 | 每天一次 |
| Mini-game（参与） | +5 💰 | 每场 |
| Mini-game（获胜/高分） | +15~50 💰（随分数） | 每场 |
| 照顾动作（每次） | +2 💰 | 每次操作 |
| 7天连续签到 bonus | +100 💰 | 每7天 |
| 繁殖成功奖励 | +200 💰 | 一次性 |

**活跃用户日均收入估算：**
> 签到 20 + 3场游戏均值 90 + 照顾 20 ≈ **130 💰/天**

### 9.2 商店价格表

| 商品 | 价格 | 备注 |
|------|------|------|
| 正餐 × 5 | 15 💰 | 日常消耗 |
| 零食 × 3 | 20 💰 | 日常消耗 |
| 药品 | 20 💰 | 紧急用 |
| 洗澡道具 | 15 💰 | 可选 |
| 玩具（加心情） | 25 💰 | 非必需 |
| 全状态回满道具 | 60 💰 | 急用 |
| 神秘蘑菇（盲盒） | 25 💰 | 赌博感 |
| 外观配件（基础） | 80 💰 | 收集向 |
| 外观配件（稀有） | 200 💰 | 收集向 |
| 繁殖道具 | 120 💰 | 解锁繁殖 |
| **复活道具** | **888 💰** | 极度 edge case |

**经济平衡分析：**
- 复活道具 888 💰 ÷ 130 日均 ≈ **约7天积累可购买**
- 7天是宠物生命周期的合理比例，玩家会认真考虑是否值得
- 繁殖道具 120 💰 ≈ 1 天积累，门槛合理
- 外观稀有配件 200 💰 ≈ 1.5 天，收集向不影响核心玩法

## 十、死亡 & 继承系统

### 10.1 死亡触发

- 健康值归零 → 播放死亡动画 → 进入「继承选择界面」

### 10.2 继承选择界面

玩家可选择：

**选项 A：重新开始（选择新宠物）**
- 完全重新选一只新宠物，从蛋开始
- 保留所有货币和道具库存

**选项 B：基因继承（继续上一只的血脉）**
- 新宠物外观基于上一只的「进化路线」继承基因（可能有细微变化）
- 从幼年阶段开始（跳过孵蛋）
- 保留所有货币和道具库存
- 隐藏彩蛋：代数越高，可能出现「祖传特征」

**选项 C：复活（消耗 888 💰）**
- 宠物从濒死状态原地复活，回到健康 50
- 不进化也不倒退，保持当前形态
- 极贵，供极度不舍得的玩家使用

## 十一、繁殖系统

### 11.1 触发条件

- 宠物达到成年阶段
- 玩家持有「繁殖道具」（商店购买，120 💰）
- 自愿触发，不强制

### 11.2 三种配对方式

#### 方式 A：路过的陌生宠物（系统生成）
- 系统随机生成一只「路过的宠物」拜访
- 玩家可接受或拒绝
- 后代外观随机混合双亲特征

#### 方式 B：时间胶囊
- 宠物成年后，玩家可主动「存入时间胶囊」保留当前基因
- 宠物死亡后，可用时间胶囊生成后代（延迟繁殖）
- 适合玩家对某只特别满意时留存

#### 方式 C：玩家互配（需双方确认）
- 玩家 A 在 App 内生成「宠物基因码」（6位随机码）
- 玩家 B 输入该码，发出配对申请
- 玩家 A 收到通知，确认后触发繁殖
- 双方各得一只后代（后代不同，但都含双亲基因）
- 双方均获得繁殖成功货币奖励

### 11.3 繁殖结果

- 后代取代当前宠物（自动进入孵蛋界面）
- 后代外观：基于双亲的颜色/配件/进化路线生成变体
- 后代代数 +1（界面可显示「第X代」）
- 触发「繁殖成功」货币奖励 +200 💰 + 太阳花蜜道具

## 十三、账号 & 数据系统

### 13.1 账号

- 使用 Supabase Auth（邮箱登录 + 可选 Google OAuth）
- 每个账号对应一只宠物（MVP）
- 数据云端实时同步

### 13.2 云端存储数据结构（草稿）

```
user_pets {
  id, user_id, pet_type,           // 宠物基本信息
  stage, evolution_form,            // 成长阶段
  hunger, mood, health,             // 状态值
  last_updated_at,                  // 用于计算离线时间损耗
  generation, parent_genes,         // 繁殖系统
  care_mistakes, training_score,    // 进化路线追踪
  currency, inventory,              // 货币和道具
  sleep_start, sleep_end,          // 睡眠设定
  created_at
}
```

### 13.3 离线计算

- 每次打开 App，根据 `last_updated_at` 和当前时间计算「离线期间」的状态消耗
- 睡眠时段内使用慢速消耗，其他时间正常消耗
- 最多计算 48 小时的离线损耗（超过则视为「最大损耗」，避免一次性死亡）

## 十六、UI 导航结构

### 16.1 底部 4 Tab 导航

```
🐾 宠物  │  🎒 背包  │  🛒 商店  │  📖 家谱
```

#### 命名决策：第四个 Tab 叫「家谱」

功能内容决定了名字：

| 功能模块 | 说明 |
|---------|------|
| 宠物血统树 | 当前宠物 + 历代祖先形态记录（代数、进化路线、存活时长） |
| 繁殖历史 | 配对记录，后代列表 |
| 成就徽章 | 连续签到、养成次数、珍稀进化等成就 |
| 睡眠时间设定 | 设置宠物的睡觉/起床时间 |
| 解锁内容 | 已获得的背景主题 |
| 账号 | 头像、用户名、登出 |

> 「家谱」直接命中核心差异化功能（血统/繁殖），游戏感强，比「图鉴」更准确，比「我的」更有灵魂。  
> 桌面端可将 4 tab 改为左侧边栏。

### 16.2 每个 Tab 完整内容清单

#### 🐾 宠物（主界面）
- 宠物 Sprite 动画区（全屏主体，宠物一直在动）
- 三条状态进度条：饱食 / 心情 / 健康

- 便便图标（有便便时出现，点击清理）
- Mini-game 入口（玩耍按钮进入）

#### 🎒 背包
- 道具网格（分类 Tab：全部 / 食物 / 药品 / 特殊）

- 特殊道具有效果说明tooltip

- **解锁内容**：背景主题已拥有列表

- **账号区**：头像/用户名、登出按钮
